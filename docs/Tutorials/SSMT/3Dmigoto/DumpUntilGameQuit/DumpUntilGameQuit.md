# 🗂️ Dump 导致游戏卡死怎么办

## 🚫 问题描述

在使用 **3Dmigoto** 的 **Hunting** 模式进行 **Dump** 时，有时会导致游戏直接卡死或崩溃。

## 🔍 原因分析

- 这取决于你当前的 **Hunting** 界面有多少个 **IB**。
- **IB** 越多，**Dump** 的文件越多。
- **Unity** 游戏如果 **Dump** 很多的话，会导致游戏卡住断线直到 **Dump** 完重连为止。
- **UE** 游戏如果 **Dump** 很多的话，会一直 **Dump** 直到 120 秒触发 **UE** 崩溃自检自动卡退游戏。

## 🛠️ 解决方案

1. **控制 IB 数量**：记得在 **Dump** 之前，让 **IB** 的总数尽可能小，控制在 100 个以内最好。
2. **清空累计 IB**：按小键盘 `+` 号键可以把累计的 **IB** 清空，每次 **Dump** 之前都要按的，不然 **IB** 数值会很多且容易报错。
3. **优化视角**：**Dump** 时尽量摄像机朝天，同时让你要 **Dump** 的物品出现在摄像机里，这样能减少很多 **IB**，一般的游戏都是天空上不会放很多物体。
4. **开启 Symlink 特性**：可以开启 **Symlink** 特性来减小 **Dump** 文件的大小以及减少 **Dump** 时间。

> 💡 **提示**：合理控制 **Dump** 范围和数量，可以有效避免游戏卡死问题。
